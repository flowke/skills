## 当前已知约束

- 其他流程与其他项目一致，可参考超级教研的发布协作方式。
- 发测试：通常把当前工作分支或用户指定分支合并到 `dev` 分支。
- 发灰/发灰度：把目标分支合并到 `gray` 分支。
- 发灰和发测试完成代码合并后，应直接把对应发布页展示出来，方便用户点击进入网页发布。
- 发线上不是直接合并代码到 `master`，而是：
  1. 创建 MR
  2. 拉取最新 tag
  3. 升级 tag 版本
  4. 推送新 tag
- 用户不能直接执行 `push master`，因此线上流程可能需要依赖 GitLab API、GitLab CLI，或其他具备受控权限的自动化方式。
