# 仓库页面返回偏好

- 作用范围：**工作模块下的仓库相关对话**。
- 当用户说“仓库页面”的时候，优先理解为：**给出当前正在讨论或当前已确认仓库的页面地址**。
- 输出方式：**直接输出地址即可**。
- 不需要额外执行打开网页、补充解释或给出多余选项，除非用户另外要求。

# 说明

- 这是一个 **work-scoped / repo-aware** 偏好：适用于“工作”语境中的仓库，不只限于 `edu-content-studio`。
- 具体地址应根据当次对话里已经锚定的仓库身份来确定，例如仓库名、remote、README、目录名等。
- 如果仓库身份尚未锚定，应先确认或推断仓库身份，再返回对应页面地址，避免把某一个仓库地址误用于其他仓库。
