# 知音楼给同事发消息流程

## 适用场景

当需要给某位同事发送知音楼私聊消息时使用。当前记录的示例同事为：杜涔涔，工号 `113166`，见 `topics/同事/杜涔涔.md`。

## 推荐流程

1. **确认接收人信息**
   - 优先确认工号；如果只有姓名，先补全到工号。
   - 这次记录中的同事工号：`113166`。

2. **确认 yach-message skill 可用**
   - 使用 skill：`$yach-message`
   - 该 skill 用于主动发送知音楼消息，不用于读文档。

3. **检查配置**
   - 先查看生效配置：
     - `python3 skills/yach-message/scripts/resolve_yach_config.py --summary`
   - 确保存在可用的 `appkey`、`appsecret` 和 `base_url`。

4. **优先先预览，再发送**
   - 如果不确定收件人或内容，先用 `--dry-run` 预览。
   - 私聊发送脚本：
     - `python3 skills/yach-message/scripts/send_single_message.py`

5. **按工号发送私聊消息**
   - 推荐参数：
     - `--to-work-code '113166'`
     - `--msgtype text` 或 `--msgtype markdown`
     - `--content '...'`
   - 示例：
     ```bash
     python3 skills/yach-message/scripts/send_single_message.py \
       --to-work-code '113166' \
       --msgtype text \
       --content '你好，测试消息'
     ```

6. **发送后记录结果**
   - 若消息发送成功，把发送时间、内容摘要和结果记录到本模块的 `logs/` 中。

## 说明

- 如果以后需要发群消息，用 `send_group_message.py`。
- 如果以后需要“以员工身份发送”，再考虑实验版脚本，但默认仍推荐机器人私聊方式。


## MR 链接发送模板

当场景是“发线上后，把已创建的 MR 链接发给默认协作者杜涔涔”时，默认使用 `text` 消息，模板如下：

```text
涔涔, 帮合并下: {mr_url}, 盼回复
```

其中 `{mr_url}` 替换为实际已创建的 MR 链接。

### 推荐命令示例

```bash
python3 skills/yach-message/scripts/send_single_message.py   --to-work-code '113166'   --msgtype text   --content '涔涔, 帮合并下: https://git.100tal.com/xxx/merge_requests/123, 盼回复'   --dry-run
```

### 说明

- 当前默认不附带 MR 标题。
- 当前默认不使用 markdown。
- 当前默认直接称呼“涔涔”，并以“帮合并下”“盼回复”为固定措辞。


## 临时改发给其他人

默认接收人工号是 `113166`（杜涔涔）。

如果某次发线上需要临时改发给其他人，优先覆盖接收人工号，而不是改全局规则。

推荐方式：

```bash
python tools/create_mr_and_notify.py   --repo /path/to/repo   --title "feat: xxx"   --notify-work-code '076429'   --notify-name '张三'   --notify-dry-run
```

规则如下：

- 只传 `--notify-work-code`：使用兜底文案 `帮合并下: {mr_url}, 盼回复`
- 同时传 `--notify-name`：使用文案 `{name}, 帮合并下: {mr_url}, 盼回复`
- 传 `--notify-content`：完全按用户给定模板发送
