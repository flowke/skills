# Document Structures

在需要显式展示或重建文档结构时，按场景选择最小骨架；不要一开始就展开全部栏目。

```md
- 当前最关键：
- 其他：

## 下一步聚焦
1. 
   - 建议：
```

## 选择规则

- 主题还很散时，先用“通用骨架”。
- 已明确是需求讨论时，切到“产品/需求讨论骨架”。
- 已明确是架构或取舍讨论时，切到“技术方案讨论骨架”。
- 已明确是在聊模糊方向或机会点时，切到“泛化想法收敛骨架”。
- 用户没有要求完整模板时，只展示当前真正需要的栏目。
