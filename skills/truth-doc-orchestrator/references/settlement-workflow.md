# Settlement Workflow

适用于“活跃意图已经完成，需要把结果沉淀为当前真相”的场景。

## 核心原则

- 沉淀的对象不是过程文档本身，而是新确认的事实、已兑现的意图与更新后的当前真相
- 活跃意图目录用于承载过程与上下文，不应长期承担当前真相
- 沉淀结果应尽量压缩成可追溯、可导航、可复用的单文件结果
- 若内容跨模块稳定复用，应晋升到 `shared-truths/`

## 沉淀动作

1. 从 `10-事实基础.md` 与 `30-行为语义.md` 中提取新确认的事实真相
2. 将稳定事实写回 `facts/protocols/`、`facts/domain/`、`facts/ui-ux/`、`facts/references/`
3. 从 `00-意图总览.md`、`20-意图图谱.md`、`30-行为语义.md` 中提取已兑现的意图真相
4. 从 `40-工程落地.md` 中提取长期有效的工程边界与依赖事实
5. 从 `50-验证与回归.md` 中提取验证结论与假设验证结果
6. 更新 `facts/10-当前能力地图.md`
7. 判断是否需要晋升到 `shared-truths/`
8. 生成 `intents/settled/<意图ID>-<意图名>.md`
9. 更新 `intents/settled/00-沉淀索引.md`
10. 更新 `00-模块导航.md`
11. 从 `intents/active/` 移除原完整目录

## 最小产出

至少完成以下 5 件事：

- [ ] 新确认的事实真相已写回 `facts/`
- [ ] `facts/10-当前能力地图.md` 已反映本次结果
- [ ] 必要时已晋升到 `shared-truths/`
- [ ] `intents/settled/` 中已有单文件沉淀结果
- [ ] 原完整意图目录已从 `intents/active/` 移除

## 固定输出结构

建议沉淀结果按以下结构输出：

```md
# <模块名> <意图ID>-<意图名> 沉淀结果

## 1. 新确认的事实真相
- 文档：
- 内容：

## 2. 已兑现的意图真相
- 意图点：
- 结果说明：

## 3. 假设验证结果
- 假设：
- 结果：已证实 / 已证伪 / 延后确认
- 影响：

## 4. 当前真相更新
- 能力项：
- 当前说明：
- 关联入口：

## 5. 共享真相晋升
- 候选内容：
- 晋升原因：
- 目标位置：

## 6. 原活跃意图目录处理
- 原路径：
- 处理结果：已移除
```

若某项为空，也应显式写“无”或“本次无需更新”。

## 不要这样做

- 不要把当前真相只留在 `active/` 目录
- 不要让沉淀文件继续承接新范围
- 不要跳过 `facts/10-当前能力地图.md` 的更新
- 不要让 `shared-truths/` 长期缺失真正跨模块复用的稳定真相
