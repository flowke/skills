# Diagram Guidelines

Mermaid 图是正式表达工具，用来提升“可讨论、可复核、可追溯”的程度。

## 什么时候应该画图

满足任意一条时，优先考虑 Mermaid：

- 角色多于 2 个
- 状态多于 3 个
- 有明显时序关系
- 有分支、回退、补偿、重试逻辑
- 存在异步协作
- 纯文字表达已经开始绕
- 目标结构本身更适合图而不是表

## 常用图种

### `flowchart`
适合：
- 目标路径
- 模块结构
- 分支逻辑
- 处理流程

### `sequenceDiagram`
适合：
- 角色交互
- 接口时序
- 前后端协作
- 编排调用链

### `stateDiagram-v2`
适合：
- 状态机
- 生命周期
- 状态转移条件
- 异常与回退状态

### `erDiagram`
适合：
- 领域对象关系
- 数据实体关系
- 聚合结构

### `journey`
适合：
- 用户路径
- 多步骤体验流

## 图的书写规则

- 图必须服务于某个明确问题
- 图与正文必须一致
- 图中节点命名要稳定
- 图太复杂时，拆成多个小图
- 图后应补一句“这张图主要表达什么”
- 若图中存在假设或待确认项，应显式标注

## 不推荐画图的情况

- 只有单步线性逻辑
- 图比正文更绕
- 图无法维护
- 图与意图、行为或事实没有直接关系
- 图只是为了“显得完整”
