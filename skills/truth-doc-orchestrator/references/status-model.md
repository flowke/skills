# Status Model

## 意图状态

允许值：
- `构思中`
- `事实已立`
- `意图已定`
- `落地中`
- `验证中`
- `校准中`
- `带漂移推进`
- `已沉淀`

## 推荐流转

主路径：
`构思中 -> 事实已立 -> 意图已定 -> 落地中 -> 验证中 -> 已沉淀`

漂移分支：
`构思中 / 事实已立 / 意图已定 / 落地中 / 验证中 -> 校准中`

若漂移可隔离：
`校准中 -> 带漂移推进 -> 原阶段或验证中`

若漂移已完成回收：
`校准中 -> 原阶段`

## 状态进入条件

### 构思中
- 目标尚未稳定
- 事实仍在收集
- 关键待确认项较多

### 事实已立
- 支撑本次意图的关键事实已被识别和整理
- `facts/` 已能承接当前已知现实

### 意图已定
- `00-意图总览.md` 已明确目标、范围、验收口径
- `10-事实基础.md` 已说明关键事实依赖
- `20-意图图谱.md` 已能表达目标结构

### 落地中
- `30-行为语义.md` 与 `40-工程落地.md` 已明确
- 已进入实施

### 验证中
- 实施已完成或进入可验证阶段
- `50-验证与回归.md` 已可执行

### 校准中
- 已发现真相与实现或运行现实存在明确漂移
- 漂移已被记录，并且需要优先完成分级、范围划定或处理路径选择
- 当前阶段不能继续按“稳定事实”假设向下推进全部内容

### 带漂移推进
- 已识别漂移并完成分级
- 影响范围与当前受信切片已明确
- 临时假设、限制条件与下一步校准动作已写出
- 仅在受信切片内继续推进，不能将整体误标为稳定完成

### 已沉淀
- 验证闭环完成
- `intents/settled/` 中已有单文件结果
- `facts/10-当前能力地图.md` 已更新
- `00-模块导航.md` 已更新
- 原完整意图目录已从 `intents/active/` 移除

## 事实文档状态

允许值：
- `草拟中`
- `已确认`
- `待更新`
- `存在漂移`
- `仅临时可用`

## 推荐流转

稳定路径：
`草拟中 -> 已确认 -> 待更新 -> 已确认`

漂移路径：
`已确认 -> 存在漂移 -> 仅临时可用 -> 已确认`

### 草拟中
- 事实仍在整理
- 结论尚不稳定

### 已确认
- 已有足够事实来源支撑
- 当前可以被下游意图依赖

### 待更新
- 当前事实可能因新意图完成、外部变化或实现调整而失效或不完整

### 存在漂移
- 当前文档中的部分结论已与实现或运行现实不一致
- 影响范围已知，但尚未完成修正
- 不应再被下游当作全量稳定真相引用

### 仅临时可用
- 已识别受信切片
- 仅部分事实仍可被依赖
- 超出边界的部分必须配合假设、待确认项或校准计划使用

## 更新优先级

1. 记录漂移及影响范围
2. 修正事实文档正文
3. 修正文档状态
4. 更新当前能力地图
5. 生成沉淀结果文件与沉淀索引
6. 更新模块导航
7. 移除原活跃意图目录
