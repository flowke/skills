# 第 11 章：研究进阶：麦肯锡分析师的“资讯料理法”

## Core Idea
高质量研究像做菜：备菜是输入资讯，烹饪是处理资讯，摆盘是输出资讯；分析师的核心能力在于假设驱动、资讯拣选、金字塔提炼和清晰表达。

## Frameworks Introduced
- **资讯料理法**：备菜 → 烹饪 → 摆盘。
  - When to use: 组织任何研究任务、行业报告、商业分析或投资分析。
  - How: 先明确假设和信息源，再提炼结构化观点，最后输出有逻辑的表达。
- **信息过载三步法**：做出假设 → 拆解假设 → 验证子假设。
  - When to use: 面对海量资料、不知道从何查起。
  - How: 用 MECE 拆成可验证子问题，再为每个子问题找指标或证据。
- **观点极化三步法**：拓宽输入渠道 → 做好资讯拣选 → 理性赋权资讯。
  - When to use: 不同信息源观点冲突严重时。
- **金字塔四步法**：罗列资讯 → 寻找规律 → 提炼要点 → 归纳核心。
  - When to use: 从资料中生成有深度、有逻辑的结论。

## Key Concepts
- **备菜**：资讯输入，包括找资料、筛选资料和验证事实。
- **烹饪**：资讯处理，包括发现规律、提出洞察和搭建逻辑。
- **摆盘**：资讯输出，包括结构、表达和图文呈现。
- **MECE**：相互独立、完全穷尽的拆解原则。
- **事实判断**：可验证的事实陈述。
- **价值判断**：基于事实但包含观点、偏好或推论的判断。
- **系统 1 / 系统 2**：直觉快速反应与理性深度思考。

## Mental Models
- **不要大海捞针，要假设驱动**：先提出可证伪假设，再找证据。
- **观点要有事实支撑**：价值判断可以参考，但要追溯到经过验证的事实。
- **提炼不是总结**：总结压缩原文，提炼要产生新的认知；多问“那又怎样”。
- **输出是认知产品的包装**：好报告要让别人快速看到结论、逻辑和证据。

## Anti-patterns
- **无假设搜资料**：会陷入信息过载，研究效率极低。
- **只看支持自己观点的信息**：会被算法推荐和确认偏误困住。
- **把事实和观点混在一起**：导致结论无法验证。
- **报告像资料库**：缺少金字塔结构，读者看不到主线。

## Key Takeaways
1. 研究过程可以类比为备菜、烹饪、摆盘。
2. 信息过载用假设驱动解决。
3. 观点极化要拓宽渠道、拣选资讯、理性赋权。
4. 金字塔原理帮助从资料提炼核心观点。
5. 好研究既要有深度，也要有逻辑和表达。
