# 第 2 章：可行性：既然知道会输，那为何要“下注”

## Core Idea
导入期行业最重要的问题不是“市场多大”或“估值多少”，而是商业模式是否跑得通：是否有人买、是否能赚钱。

## Frameworks Introduced
- **商业模式六要素重组**：商业模式可理解为生产力三要素与生产关系三要素的排列组合。
  - When to use: 拆解一个新业务、新赛道或新公司到底靠什么创造和分配价值。
  - How: 分别看劳动者、劳动资料、劳动对象，以及所有制、关系位置、分配方式。
- **销售可行性 + 利润可行性**：可行性判断要拆成“卖得出去”和“赚得到钱”。
  - When to use: 分析早期赛道、创业项目、一级市场标的。
  - How: 先验证需求真实存在，再验证单位经济模型是否有正向可能。
- **时间对标法 / 空间对标法**：验证需求时向过去和成熟市场找参照。
  - 时间对标: 过去是否有相似需求或替代方案。
  - 空间对标: 海外、其他城市、成熟市场是否已有成功案例。
- **UE 模型**：用单位经济模型判断单个用户、订单、车辆、门店或设备是否能赚钱。

## Key Concepts
- **商业模式**：企业组织资源、满足需求并获取利润的方式，不存在唯一标准答案。
- **销售可行性**：目标用户是否真的愿意为解决方案付费或迁移。
- **利润可行性**：收入、成本、频次、弹性和规模化后是否能支持盈利。
- **频次**：用户重复购买或使用的频率，影响收入天花板。
- **弹性**：价格变化对需求的影响程度，影响提价能力。
- **标准化程度**：成本能否被流程化、规模化和复制化降低。

## Mental Models
- **先问“有没有旧需求”**：成功商业模式往往不是凭空创造需求，而是用新方式满足旧需求。
- **不要只看热闹，要看单位账**：概念火不代表商业模式可行，UE 模型能暴露补贴和亏损依赖。
- **创新有层级**：基础层创新往往先发生，中间层、应用层创新随后扩散。

## Anti-patterns
- **把融资能力当商业模式**：靠融资补贴增长不等于盈利模型成立。
- **只验证需求，不验证利润**：能卖出去但赚不到钱，仍不是可持续商业模式。
- **盲目照搬海外案例**：空间对标需要检查制度、文化、收入水平、供给条件是否相似。

## Key Takeaways
1. 导入期最重要的是判断商业模式可行性。
2. 可行性必须同时看销售和利润。
3. 时间对标和空间对标适合验证需求真实性。
4. UE 模型适合验证利润模型和规模化潜力。
5. 对创新要区分基础层、中间层和应用层，避免把应用热潮误判成底层变革。
