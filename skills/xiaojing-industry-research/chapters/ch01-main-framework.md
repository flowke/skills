# 第 1 章：主框架：行业不同发展阶段的分析重点

## Core Idea
行业研究首先不是“找资料”，而是界定研究对象并判断行业所处的生命周期阶段；不同阶段要抓不同重点，否则容易用错分析工具。

## Frameworks Introduced
- **行业定义三参照**：当行业边界模糊时，先参考《国民经济行业分类》、证监会行业分类、申万行业分类，再根据研究目的重定义边界。
  - When to use: 研究对象跨产品、跨技术、跨上下游，或报告口径不一致时。
  - How: 先列出不同分类口径，再说明本次研究采用哪个口径以及为什么。
- **渗透率生命周期划分**：用渗透率而非笼统“萌芽/成长/成熟”判断行业阶段。
  - When to use: 判断新兴行业从概念走向主流、或成熟行业是否接近天花板。
  - How: 15%–20% 常作为导入期与成长期分界；35%–40% 常作为成长期与成熟期分界。
- **阶段—重点映射框架**：导入期看可行性，成长期看规模性，成熟期看防守性和盈利性，衰退期看替代风险和第二增长曲线。

## Key Concepts
- **行业研究对象**：不是天然给定的标签，而是围绕研究目的划定的一组需求、供给、竞争者和产业链关系。
- **产业生命周期**：行业从导入、成长、成熟到衰退的演进过程。
- **渗透率**：某产品或服务在目标市场中的采用比例，可作为阶段判断的关键指标。
- **基本面**：行业赚钱能力的底层结构，包括商业模式、市场规模、护城河和竞争格局。
- **景气度**：行业短中期业绩增速和经营状态的高频表现。

## Mental Models
- **先定位，再分析**：不要一上来就搜数据；先判断行业处于哪一段生命周期，再决定该看什么。
- **框架是地图，不是答案**：框架帮助你避免遗漏，但每个行业仍需根据实际供需、政策和技术演化修正。
- **静态判断必须动态追踪**：一次性的生命周期判断只是起点，外部因素和景气度会不断改变行业状态。

## Anti-patterns
- **用成熟行业逻辑分析导入期行业**：过早追问利润率、PE 等指标，可能错过真正关键的商业模式验证。
- **只看行业名字不看边界**：同一行业在不同分类体系中范围不同，直接比较数据容易口径错配。
- **把生命周期当定性形容词**：如果没有渗透率、增速、供需等指标支撑，“高成长”“成熟”只是口号。

## Key Takeaways
1. 行业研究的第一步是定义行业边界，而不是搜集资料。
2. 渗透率是判断产业生命周期阶段的实用指标。
3. 生命周期阶段决定分析重点：可行性、规模性、防守性、盈利性依次成为核心问题。
4. 估值、外部因素和景气度贯穿所有阶段。
5. 对行业要持续动态跟踪，不能把一次研究结论永久化。
