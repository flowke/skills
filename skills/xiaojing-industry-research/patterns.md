# Patterns — 《如何快速了解一个行业》

## Pattern 1: 行业研究八步法
**When to use**: 需要快速理解一个新行业、写行业报告、判断投资赛道或做商业分析。

**How**:
1. 定义行业边界和研究口径。
2. 判断产业生命周期阶段。
3. 导入期看商业模式可行性。
4. 成长期看市场规模和增长空间。
5. 成熟期看护城河和竞争格局。
6. 按阶段选择估值逻辑。
7. 识别 PEST 外部因素。
8. 建立景气度指标体系持续跟踪。

**Trade-offs**: 快速但依赖输入数据质量；适合作为第一轮结构化研究，不替代深度尽调。

## Pattern 2: 商业模式可行性评估
**When to use**: 早期行业、创业项目、新产品、新商业模式。

**How**:
1. 判断销售可行性：需求是否真实、用户是否愿意迁移或付费。
2. 用时间对标法寻找历史类似需求。
3. 用空间对标法寻找成熟市场案例。
4. 判断利润可行性：频次、价格弹性、成本标准化程度。
5. 搭建 UE 模型验证单位经济是否可持续。

**Trade-offs**: 对标法有迁移风险；UE 模型对假设敏感，必须动态修正。

## Pattern 3: 市场规模测算
**When to use**: 判断行业能否支撑大公司、投资空间或业务进入价值。

**How**:
1. 明确测算口径：TAM、SAM 或 SOM。
2. 需求导向：用户数 × 频次 × 单价 × 渗透率/替代率。
3. 供给导向：从产能、渠道、资源、监管瓶颈反推。
4. 供需匹配：用成熟市场、局部样本或比例关系交叉验证。
5. 输出关键假设和敏感性。

**Trade-offs**: 数字看似精确但本质是基于假设；必须披露口径和变量。

## Pattern 4: 护城河拆解
**When to use**: 判断成熟行业或龙头企业能否抵御竞争和替代。

**How**:
1. 识别资源垄断：劳动力、土地、资本、技术、数据。
2. 识别网络效应：政府、同行、供应商、客户关系。
3. 判断这些优势如何带来更高收入或更低成本。
4. 检查外部因素是否会削弱护城河。
5. 对比替代品是否能绕开原有壁垒。

**Trade-offs**: 护城河经常被高估；必须动态评估。

## Pattern 5: 竞争格局与盈利性分析
**When to use**: 分析成熟行业、周期行业、制造业、资源品或消费龙头。

**How**:
1. 横向看同行：集中度、份额变化、产能周期。
2. 纵向看产业链：上下游集中度、供需、议价能力。
3. 判断利润分配由谁主导。
4. 跟踪产能利用率、价格、库存、毛利率。
5. 判断是否处于格局改善或恶化阶段。

**Trade-offs**: 格局变化可能慢但影响大；周期顶部容易误判成长期。

## Pattern 6: 生命周期估值判断
**When to use**: 判断一个行业或公司贵不贵、估值逻辑是否成立。

**How**:
1. 判断生命周期阶段。
2. 投资者侧看流动性、利率、风险偏好。
3. 资产侧看赔率：规模性、盈利性。
4. 资产侧看概率：可行性、防守性。
5. 匹配阶段逻辑：故事、增长、龙头进阶、第二增长曲线或价值陷阱。

**Trade-offs**: 估值高度受宏观和情绪影响；低估值不必然安全。

## Pattern 7: 景气度指标体系
**When to use**: 需要高频跟踪行业、预判业绩或验证投资假设。

**How**:
1. 找到行业利润或收入的核心驱动变量。
2. 选择高频且能反映基本面的指标。
3. 从产业链纵向找核心驱动环节。
4. 从行业横向选择价格、库存、销量、开工率等指标。
5. 定期复盘指标和财报之间的关系。

**Trade-offs**: 高频数据噪声大；指标逻辑会随行业变化失效。

## Pattern 8: 资讯料理法
**When to use**: 面对复杂研究问题、资料过多或观点冲突。

**How**:
1. 备菜：假设驱动收集资讯，区分事实判断和价值判断。
2. 烹饪：罗列资讯、寻找规律、提炼要点、归纳核心。
3. 摆盘：用金字塔结构输出结论、逻辑和证据。

**Trade-offs**: 需要持续练习；过早假设可能带来确认偏误，因此要主动找反证。

## Pattern 9: AI-assisted Research Workflow
**When to use**: 快速熟悉行业、生成研究提纲、整理资料和提高输出效率。

**How**:
1. 让 AI 生成行业概览和关键词清单。
2. 让 AI 拆解研究假设和数据需求。
3. 用 AI 汇总资料和构建对比表。
4. 用 AI 生成报告大纲、反方观点和检查清单。
5. 回到原始来源验证事实、数字和政策。

**Trade-offs**: AI 会幻觉且可能过时；不能替代事实核查和专业判断。
