# Document Location

默认把 `discuss-and-distill` 产出的讨论文档放到固定目录：`docs-discuss-and-distill/`。

## Default Rules

- 固定目录：`docs-discuss-and-distill/`
- 默认单位：每个主题一份文档
- 默认命名：`YYYY-MM-DD-<中文主题>.md`
- 同主题后续讨论：继续维护原文档
- 明显切换新主题：新建文档

## Naming Guidance

文件名优先体现主题，而不是会话本身；**优先使用简洁明确的中文标题**。

推荐示例：
- `2026-03-19-内部工具范围收敛.md`
- `2026-03-19-认证重构方向.md`
- `2026-03-19-产品定位讨论.md`

避免过于宽泛或无信息量的名字，例如：
- `notes.md`
- `discussion.md`
- `today.md`
- `2026-03-19-聊聊这个.md`

## Fallback Rule

满足下面情况时，可以不用中文文件名，改用英文或 slug：
- 用户明确要求英文文件名
- 当前项目已有稳定的英文命名约定
- 文件系统、工具链或协作环境对中文文件名不友好

## Reuse Rule

满足下面条件时，优先复用已有文档，而不是新建：
- 讨论主题没有变化，只是在继续收敛
- 当前文档中的未决问题仍然是本轮讨论重点
- 用户明确表示“继续上次那个”

## New File Rule

满足下面条件时，优先新建文档：
- 讨论目标明显切换
- 原文档已经收敛完成，当前是在开新主题
- 用户明确要求新开一份

## Override Rule

如果用户明确指定了目标路径、目标文件名或项目内既有文档位置，优先遵循用户指定，不强制写入 `docs-discuss-and-distill/`。
