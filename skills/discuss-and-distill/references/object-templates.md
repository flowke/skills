# Object Templates

当 `discuss-and-distill` 需要创建或维护 `intake / task / module / topic / knowledge` 等对象时，优先参考这里的**轻量模板建议**。

注意：
- 这里提供的是**推荐结构**，不是强制 schema。
- 目标是帮助产物保持可读、可继续推进、可回写，而不是把系统做成厚重的管理模板。
- 如果对象已经有稳定结构，优先延续现有结构，而不是为了模板强行重写。

## General Rules

- 稳定对象优先写清“当前已经成立什么”，不要把执行细节压进 `current-truth.md`。
- 执行对象优先写清“现在要做什么、挂在哪、状态如何”，不要把 task 写成冗长日志。
- 共享知识优先写清“这是什么、适用于哪里”，不要强制统一所有知识文档格式。
- 模板应默认轻量；只有当场景明确需要时，再补更重章节。

## 1. intake / note.md

适用场景：
- 资料先入队，尚未正式创建 task

### 推荐最小字段
- `原始描述`
- `来源`
- `当前状态`
- `备注`

### 可选字段
- `候选挂接对象`
- `是否已转 task`
- `相关链接`

### Minimal Template

```md
# Intake Note

- 原始描述：
- 来源：
- 当前状态：待处理
- 备注：

## 相关资料
- 
```

## 2. tasks / task.md

适用场景：
- 承接一次执行活动

### 强必有
- `任务类型`
- `挂接对象`
- `当前状态`

### 推荐字段
- `当前目标`
- `输入依据`
- `下一步动作`
- `输出去向`

### 可选字段
- `来源 intake 项`
- `当前阻塞`
- `相关子任务`

### Minimal Template

```md
# Task

- 任务类型：
- 挂接对象：
- 当前状态：

## 当前目标
- 

## 输入依据
- 

## 下一步动作
- 

## 输出去向
- 
```

## 3. tasks / subtask-*.md

适用场景：
- 父 task 需要拆分多个轻量子任务

### 强必有
- `任务类型`
- `挂接对象`
- `当前状态`

### 推荐字段
- `子任务目标`
- `与父任务关系`
- `完成后回写位置`

### Minimal Template

```md
# Subtask

- 任务类型：
- 挂接对象：
- 当前状态：

## 子任务目标
- 

## 与父任务关系
- 

## 完成后回写位置
- 
```

## 4. modules / topics / current-truth.md

适用场景：
- 记录稳定对象的当前真相

### 推荐最小结构
- `讨论主题`
- `当前真相`
- `待确认项`
- `未决问题`
- `下一步聚焦`

### Notes
- 模块与主题共用同一 truth 骨架即可。
- 模块如果存在局部职责拆分，可把局部真相写到 `submodule-*.md`。
- 不要把 task 执行流水直接堆进 `current-truth.md`。

## 5. modules / submodule-*.md

适用场景：
- 模块内部需要独立维护一个子模块 truth

### 推荐最小字段
- `子模块职责`
- `当前真相`
- `待确认项`
- `未决问题`
- `下一步动作`

### Minimal Template

```md
# Submodule Truth

- 子模块职责：

## 当前真相
- 

## 待确认项
- 

## 未决问题
- 

## 下一步动作
- 
```

## 6. knowledge /

适用场景：
- 记录可跨对象复用的协议、机制、约束、长期约定等知识

### 当前策略
- 不强制规定固定正文模板
- 允许按知识内容自由组织
- 推荐补充轻量元信息以便追溯与复用，但不把这些字段升级为必填 schema

### 推荐可选元信息（默认不强制）
- `来源`
- `更新时间`
- `适用范围`
- `相关对象`

### Light Example

```md
# <知识标题>

- 来源：
- 更新时间：
- 适用范围：
- 相关对象：

## 内容
- 
```

## 7. Drift Handling Notes

适用场景：
- 回归测试 task 发现 code 与 truth 不一致

### 推荐记录字段
- `漂移分类`
- `当前判断依据`
- `默认动作`
- `回写位置`

### 分类建议
- `truth 过时`
- `code 偏离`
- `暂时无法判断`

### Notes
- 先分类，再动作。
- 证据不足时，不自动改 truth，也不自动修代码。
- 如果需要把冲突升级回稳定对象层，应在对应模块 / 主题的 `待确认项` 或 `未决问题` 中留痕。
