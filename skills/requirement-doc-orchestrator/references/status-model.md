# Status Model

## 意图状态

允许值：
- `构思中`
- `事实已立`
- `意图已定`
- `落地中`
- `验证中`
- `已沉淀`

## 推荐流转

`构思中 -> 事实已立 -> 意图已定 -> 落地中 -> 验证中 -> 已沉淀`

## 状态进入条件

### 构思中
- 目标尚未稳定
- 事实仍在收集
- 关键待确认项较多

### 事实已立
- 支撑本次意图的关键事实已被识别和整理
- `facts/` 已能承接当前已知现实

### 意图已定
- `00-意图总览.md` 已明确目标、范围、验收口径
- `10-事实基础.md` 已说明关键事实依赖
- `20-意图图谱.md` 已能表达目标结构

### 落地中
- `30-行为语义.md` 与 `40-工程落地.md` 已明确
- 已进入实施

### 验证中
- 实施已完成或进入可验证阶段
- `50-验证与回归.md` 已可执行

### 已沉淀
- 验证闭环完成
- `intents/settled/` 中已有单文件结果
- `facts/10-当前能力地图.md` 已更新
- `00-模块导航.md` 已更新
- 原完整意图目录已从 `intents/active/` 移除

## 事实文档状态

允许值：
- `草拟中`
- `已确认`
- `待更新`

## 推荐流转

`草拟中 -> 已确认 -> 待更新 -> 已确认`

### 草拟中
- 事实仍在整理
- 结论尚不稳定

### 已确认
- 已有足够事实来源支撑
- 当前可以被下游意图依赖

### 待更新
- 当前事实可能因新意图完成、外部变化或实现调整而失效或不完整

## 更新优先级

1. 修正事实文档正文
2. 修正文档状态
3. 更新当前能力地图
4. 生成沉淀结果文件与沉淀索引
5. 更新模块导航
6. 移除原活跃意图目录
