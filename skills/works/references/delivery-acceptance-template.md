# Delivery Acceptance Template

用于 `deliveries/<delivery-name>/acceptance.md`，记录某一轮代码落地的验收结果。

适用场景：
- 本轮需要正式验收记录
- 用户明确要求单独验收
- 本轮改动范围较大或风险较高

目标：
- 明确本轮是否通过验收
- 记录问题项和遗留项
- 让后续是否回到 Build / Discuss 有据可依

---

# <delivery-title>

## 1. 验收结论
- 通过 / 有条件通过 / 不通过

## 2. 验收范围
- 本次验收覆盖哪些内容
- 是否按 `brief.md` 进行验收

## 3. 通过项
- 建议用 checkbox 记录已通过的验收项
- 已通过的检查
- 验收认可的部分

## 4. 问题项
- 建议用 checkbox 记录未通过项
- 偏离范围的问题
- 质量问题 / 回归问题 / 缺失项

如果没有，可以写“无”。

## 5. 遗留项
- 本轮不阻塞通过，但需要后续处理的事项
- 建议回到后续 delivery 或 discuss 的内容

## 6. 建议动作
- 直接结束本轮
- 回到 Build 修复
- 回到 Discuss 重新收敛

建议只给 1-3 条。
