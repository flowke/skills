# 素材库 / AI视频源素材生产

## 1. Topic 范围
- 页面 / 模块 / 业务域：内容生产平台素材库，现有「素材库-图书-分类」下新增 AI视频素材生产和管理能力。
- 当前状态：待确认

## 2. 需求地图

### R1. 新增 AI视频 标签页
- 原始需求：在现有素材库-图书-分类 tab 下新增 **AI视频** 标签页，点击可查看当前已上传的 AI 视频源素材，无视频则展示通用缺省页。
- 需求细化：
  - AI视频应作为图书分类下的新 tab。
  - 列表为空时使用通用缺省页。
  - 视频卡片视图同视频后期 tab 页，维持现有展示逻辑不变。
- 相关上下文：
  - C1 产品文档。
  - C2 Figma 设计稿，上传交互以设计图为准。
- 实现思路 / 方案预对齐：
  - 复用现有素材库 tab / 列表 / 缺省页能力，新增 AI视频资源类型。
  - 卡片视图优先复用视频后期 tab 的展示组件。
- 待确认：
  - AI视频 tab 是否需要权限控制或灰度。
  - AI视频资源类型在后端枚举 / 前端路由 / 查询接口中的标识。
- 当前结论：
  - 可先按“素材库图书分类下新增 AI视频资源类型”理解。

### R2. 视频上传后填写关联题目ID
- 原始需求：视频上传逻辑不变，上传后针对每一条视频资源新增「关联题目ID」字段，需手动填写关联题目ID；若未填写，则无法点击确定按钮，点击时 toast 提示：请填写视频关联题目ID。
- 需求细化：
  - 单条视频资源上传后需要绑定题目ID。
  - 未填写题目ID时，不允许确认提交。
  - toast 文案固定为：`请填写视频关联题目ID`。
- 相关上下文：
  - C1 产品文档。
  - C2 Figma 设计稿。
- 实现思路 / 方案预对齐：
  - 可在上传列表行内增加题目ID输入框，并在确定按钮提交前进行必填校验。
  - 题目ID应作为后续资源生成 / 绑定的重要业务字段，但不建议作为唯一资源标识。
- 待确认：
  - 题目ID格式、长度、是否需要校验存在性。
  - 确定按钮是整批确认还是单条确认。
- 当前结论：
  - 前端必须提供题目ID填写和必填校验；题目ID是否参与服务端校验待确认。

### R3. AI视频与 1v1 讲题模型转换
- 原始需求：视频上传且题目ID填写完成后，点击确定，调用 / 触发 1v1 讲题模型服务进行转换，并接收转换状态回调。
- 需求细化：
  - 文档中“调用1v1讲题模型接口，将视频文件+题目ID发送至模型服务”被删除线标记，说明模型调用方式可能已变化。
  - 需要接收转换状态回调。
  - 生成状态包含：待转换、转换中、转换成功、转换失败。
  - 待转换：视频上传成功，尚未触发 / 调用接口。
  - 转换中：已调用大模型接口，等待回调。
  - 转换成功：视频资源转换成功，可进行图书绑定。
  - 转换失败：列表需要有失败状态。
- 相关上下文：
  - C1 产品文档。
- 实现思路 / 方案预对齐：
  - 前端先按状态枚举展示，转换触发、回调和重试由后端 / 服务端协议确认后接入。
  - 若当前阶段接口未定，可先实现状态展示和资源ID字段预留。
- 待确认：
  - 当前最新方案是否仍由内容生产平台直接调用模型接口。
  - 回调协议、状态枚举值、错误信息字段。
  - 转换失败后是否允许手动重试。
- 当前结论：
  - 模型转换链路是当前 topic 进入 Implement 前的关键阻塞。

### R4. 唯一资源ID策略
- 原始需求：以「题目ID」作为内容生产后台与模型服务的唯一识别ID，并接收转换状态回调；但文档中也提出题目ID可能重复，倾向“内容生产定义一个唯一的资源ID”。
- 需求细化：
  - 候选唯一识别ID包括：题目ID、模型提供资源ID、内容生产侧资源ID。
  - 文档备注：题目ID担心出现重复。
  - 赵永法预留一批资源ID。
- 相关上下文：
  - C1 产品文档。
- 实现思路 / 方案预对齐：
  - 推荐由内容生产侧生成唯一资源ID，题目ID作为业务关联字段。
  - 与模型服务交互时使用资源ID作为幂等 / 回调识别 key，避免题目ID重复导致错绑。
- 待确认：
  - 资源ID由谁生成、生成时机、格式和是否批量预留。
  - 模型服务是否接受内容生产侧资源ID。
  - 资源ID和题目ID在列表、详情、图书绑定中的展示和传递规则。
- 当前结论：
  - 暂按“内容生产定义唯一资源ID，题目ID作为关联字段”作为推荐方案，但需产品 / 后端确认。

### R5. 转换失败重试机制
- 原始需求：针对网络超时 / 其他原因导致转换失败的任务进行自动重试，直至转换成功；同时文档又提到“会转换失败吗？”、“列表需要有失败状态”、“离线处理后的视频批量回显”。
- 需求细化：
  - 失败原因可能包括网络超时或其他原因。
  - 文档要求自动重试直至转换成功，但也要求列表展示失败状态。
  - 存在离线处理后的视频批量回显场景。
- 相关上下文：
  - C1 产品文档。
- 实现思路 / 方案预对齐：
  - 不建议无限自动重试；可设计最大重试次数 + 失败状态 + 人工/离线补偿。
  - 前端需要展示失败状态和可能的错误说明 / 重试入口，具体取决于产品确认。
- 待确认：
  - 是否真的“直至转换成功”无限重试。
  - 是否需要展示重试次数、失败原因、手动重试按钮。
  - 离线处理后批量回显的数据来源和刷新方式。
- 当前结论：
  - 失败重试机制仍待确认，不能直接进入实现闭环。

### R6. 批量导入题目ID
- 原始需求：视频上传列表展示「批量添加题目ID按钮」，点击可导入 excel 文件，文件字段：视频名称、题目ID；导入后根据上传的视频文件名称和文件中的视频名称完全匹配，批量填入对应题目ID。
- 需求细化：
  - Excel 字段固定为：视频名称、题目ID。
  - 匹配规则：上传的视频文件名称与 Excel 中视频名称完全匹配。
  - 若存在已上传的视频名称未匹配到对应题目ID，则 toast 提示：`{视频名称}未匹配到题目ID，请确认后重试或手动添加`。
- 相关上下文：
  - C1 产品文档。
  - C2 Figma 设计稿。
- 实现思路 / 方案预对齐：
  - 前端可先解析 Excel 并按文件名映射填充；若已有通用导入组件，优先复用。
  - 需要处理重复视频名称、Excel 重复行、空题目ID等异常。
- 待确认：
  - Excel 模板是否需要下载入口。
  - 文件名匹配是否区分后缀、大小写、空格。
  - 多个未匹配视频时 toast 是逐条提示还是汇总提示。
- 当前结论：
  - 批量导入规则主体明确，但异常细节待确认。

- 原始需求：视频列表字段包括文件名称、题目ID、格式、大小、分辨率、生成状态、创建者、操作栏；视频详情页视频信息增加题目ID字段。
- 需求细化：
  - 列表必须展示生成状态。
  - 详情页需要展示题目ID。
  - 操作栏具体动作未说明。
- 相关上下文：
  - C1 产品文档。
  - C2 Figma 设计稿。
- 实现思路 / 方案预对齐：
  - 列表字段可复用素材库已有视频字段，新增题目ID和生成状态。
  - 详情页在视频信息区域增加题目ID展示。
- 待确认：
  - 操作栏需要哪些操作：查看、删除、重试、绑定、编辑题目ID？
  - 生成状态展示样式和筛选能力。
- 当前结论：
  - 字段主体明确，操作栏和筛选/交互待确认。
